Base URL:

  • A base URL is the starting point for resolving relative URLs. It provides the context against which relative URLs are interpreted. The base URL typically includes the protocol, hostname, and path, but may exclude the fragment identifier and query parameters.
  • Example: If you have a base URL of https://www.example.com/path/to/page, this is the base against which you might resolve relative URLs.

Relative URL:

  • A relative URL specifies a resource in relation to the current document or base URL. It doesn't include the full domain or protocol but provides a path or reference relative to the current location.
  • Example: In the URL ../another/page, the ../ denotes a relative path, and it would be resolved relative to the current location.

Resolved URL:

  • A resolved URL is the result of combining a base URL with a relative URL to create a complete, fully qualified URL. The url.resolve() method in Node.js is commonly used for this purpose.
  • Example: If you have a base URL of https://www.example.com/path/to/page and a relative URL of ../another/page, resolving them would result in the resolved URL https://www.example.com/another/page.

Absolute URL:

  • An absolute URL is a complete URL that includes all the necessary components: protocol, hostname, port, path, query parameters, and fragment identifier. An absolute URL can be used independently to locate a resource.
  • Example: https://www.example.com/path/to/page is an absolute URL.

In summary, an absolute URL is a complete and standalone URL, while a relative URL provides a path or reference relative to a base URL. Resolving a relative URL against a base URL results in a fully qualified, absolute, or resolved URL. So, while "resolved URL" and "absolute URL" are not precisely synonymous, they are often used interchangeably because resolving a relative URL typically results in an absolute URL.
