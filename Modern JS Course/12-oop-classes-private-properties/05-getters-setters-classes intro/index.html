Getter and Setter Intro

Getter

Getter is like asking a question. When you want to know the value of something, you ask a question (get information).


Imagine you have a Person object:

        
    const person = {
      _name: 'John',
          
      get name() {
         return this._name;
      }
    };
          
    console.log(person.name); // Output: 'John'

In this example, get name() is like asking, "What is the name of this person?" The name getter allows you to get the value of _name from the person object.

Setter

Setter is like giving a command. When you want to change something, you give a command (set a new value).


Continuing with the Person object:

        
   const person = {
     _name: 'John',
          
      get name() {
        return this._name;
      },
            
      set name(newName) {
        this._name = newName;
      }
    };
          
    person.name = 'Jane';
    console.log(person.name); // Output: 'Jane'

Here, set name(newName) is like giving the command, "Set the name of this person to a new value." The name setter allows you to change the value of _name in the person object.

Why use Getters and Setters?

  1. Encapsulation: Getters and setters help you control how values are accessed and modified, providing a level of protection for your data.
  2. Validation: You can add checks or transformations when getting or setting values. For example, ensuring a name is always capitalized.
  3. Abstraction: Getters and setters allow you to abstract away the internal implementation details of an object.

In summary, think of getters as a way to ask questions about the state of an object, and setters as a way to give commands to change that state. They provide a clean and controlled interface for interacting with your objects.
