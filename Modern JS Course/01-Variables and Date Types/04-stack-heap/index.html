Stack

The stack is a fixed-size memory area used to store function calls and local variables. When a function is called, a new stack frame is created. This stack frame contains the function's arguments, local variables, and the return address. When the function returns, its stack frame is popped off the stack. The stack is always accessed in a last-in-first-out (LIFO) order. This means that the most recently pushed item is always the first item to be popped off.

Heap

The heap is a dynamic memory area used to store objects and functions. When an object is created, it is allocated space on the heap. When an object is no longer needed, its memory is garbage collected. The heap is accessed in a random order. This means that there is no guarantee in which order objects will be allocated or garbage collected. 

        
  // Values are stored on the stack
    let name = 'John'
    let age = 30
          
  //Reference values are stored on the heap
          
    let person = {
        name: 'Brad',
        age: 40,
    }
          
    let newName = name
    newName = 'Jonathan'
          
    let newPerson = person
    newPerson.name = 'Bradley'
          
    console.log(person.name) //Bradley

A simple analogy

Think of the stack as a stack of trays. Each time you enter a cafeteria, you take a tray from the top of the stack and place it on the table. When you leave, you remove the tray and put it back on top of the stack. It's a temporary workspace with limited capacity.


Think of the heap as a storage area where you can place objects that you want to keep for an extended period. It's like renting a storage unit where you can store items, and you need to remember to return the key when you're done.
